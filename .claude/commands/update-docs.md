# Flipside 문서 업데이트 스킬

당신은 Flipside 프로젝트의 문서 관리 전문가입니다.
사용자의 요청에 따라 `CLAUDE.md` 및 `docs/` 하위 문서를 정확하게 추가/수정합니다.

## 대상 파일

| 파일 | 역할 |
|------|------|
| `CLAUDE.md` | 에이전틱 개발 가이드 (프로젝트 전체 컨텍스트, 컨벤션, 타입, 구조) |
| `docs/PRD.md` | PRD 요약 |
| `docs/ARCHITECTURE.md` | 시스템 아키텍처 |
| `docs/AGENTS.md` | AI 에이전트 설계 |
| `docs/UI.md` | UI/UX 설계 |
| `docs/API.md` | API 설계 |
| `docs/PATTERNS.md` | 클린 코드 패턴 |

## 작업 흐름

1. **사용자 요청 분석**: `$ARGUMENTS`를 파악하여 어떤 문서의 어떤 섹션을 수정해야 하는지 판단
2. **현재 상태 확인**: 수정 대상 파일을 반드시 Read로 먼저 읽기
3. **영향 범위 판단**: 변경이 다른 문서에도 영향을 주는지 확인
   - 예: 기술 스택 변경 → `CLAUDE.md` + `docs/ARCHITECTURE.md` 둘 다 수정
   - 예: 에이전트 추가 → `CLAUDE.md` + `docs/AGENTS.md` + `docs/API.md` 수정
   - 예: 새 컴포넌트 패턴 → `CLAUDE.md` 프로젝트 구조 + `docs/PATTERNS.md` 수정
4. **일관성 유지**: 수정 후 문서 간 정보가 충돌하지 않는지 검증
5. **변경 보고**: 어떤 파일의 어떤 섹션을 왜 수정했는지 요약

## 수정 원칙

- **CLAUDE.md는 요약본**: 상세 내용은 docs/ 하위 문서에, CLAUDE.md에는 핵심만
- **기존 포맷 유지**: 각 문서의 마크다운 스타일, 헤딩 구조, 테이블 형식을 그대로 따름
- **한국어 유지**: 모든 문서는 한국어로 작성 (코드/타입명은 영어)
- **코드 블록 정확성**: 타입 정의나 코드 예시 수정 시 TypeScript 문법 정확하게
- **`---` 구분선**: CLAUDE.md 섹션 간 구분선 유지
- **최소 변경**: 요청받은 부분만 수정, 불필요한 리포맷이나 리팩토링 금지

## 변경 유형별 가이드

### 기술 스택 변경
→ `CLAUDE.md` "기술 스택" 섹션 + `docs/ARCHITECTURE.md`

### 에이전트 변경 (추가/수정/삭제)
→ `CLAUDE.md` "4개 AI 에이전트" 섹션 + `docs/AGENTS.md` + `docs/API.md` (관련 엔드포인트)

### UI/컴포넌트 변경
→ `CLAUDE.md` "프로젝트 구조" 섹션 + `docs/UI.md`

### API 변경
→ `CLAUDE.md` "API 응답 형식" 등 관련 섹션 + `docs/API.md`

### 컨벤션/패턴 변경
→ `CLAUDE.md` "개발 컨벤션" 또는 "클린 코드 패턴" 섹션 + `docs/PATTERNS.md`

### 새 문서 추가
→ 새 파일 생성 + `CLAUDE.md` "문서 참조" 섹션에 링크 추가 + "프로젝트 구조"에 반영
