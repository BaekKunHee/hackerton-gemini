# Flipside - PRD 요약 (개발자 관점)

> Gemini 3 Seoul Hackathon 2026 | Track: Gemini for Good

---

## 1. 문제 정의

### 기존 접근법이 실패하는 이유

기존 미디어 리터러시 도구들의 가정:
> "당신이 편향됐다는 걸 알면, 스스로 고치려 할 것이다"

**이 가정이 틀렸습니다:**
- 편향을 지적받으면 반성이 아닌 방어적 반응 (심리적 반발, Reactance)
- 이미 비판적 사고에 관심 있는 사람만 사용
- 정작 편향이 심한 사람일수록 안 씀

**결론**: '당신의 편향을 고쳐드립니다'는 프레임 자체가 제품을 죽이는 구조

### Flipside의 다른 출발점

**사람들은 토론에서 설득하고 싶어합니다.**

그 욕구가 있을 때:
- 마찰이 없습니다
- 상대방 논리를 이해하는 것이 이기기 위한 수단이 됩니다
- 그 과정에서 의도치 않게 비판적 사고가 작동합니다

## 2. 핵심 가치 제안

### 제품 정의
**Flipside**는 '비판적 사고 분석 플랫폼'입니다. 챗봇이 아닙니다.

철학 용어 'Steel-manning'에서 영감:
> 상대방의 논리를 가장 약한 버전이 아닌, **가장 강한 버전으로 만들어 이해하는 것**. 진짜 대화는 그때부터 시작됩니다.

### 차별점: 챗봇 vs 플랫폼

| 구분 | 챗봇 | Flipside 플랫폼 |
|------|------|----------------|
| 메인 화면 | 대화창 | 3패널 분석 대시보드 |
| 결과물 | 텍스트 답변 | 시각화된 분석 카드 |
| 에이전트 동작 | 보이지 않음 | 실시간 상태 표시 |
| 재방문 이유 | 없음 | 분석 히스토리 |
| 공유 | 불가 | 분석 카드 공유 가능 |
| 느낌 | 질문하는 곳 | 분석받는 곳 |

## 3. 타겟 사용자

### 주요 페르소나
- **토론 참여자**: 온라인/오프라인에서 토론 중인 사람
- **팩트체커**: 뉴스/정보의 신뢰성을 확인하고 싶은 사람
- **비판적 사고 학습자**: 미디어 리터러시를 향상시키고 싶은 사람

### 진입 동기 (핵심!)
- ❌ "나도 편향됐나 확인해볼까" (저항 높음)
- ✅ "상대방 논리의 허점을 찾아야지" (저항 없음)

## 4. MVP 범위 (해커톤 7시간)

### 필수 구현 (반드시 데모)

| 기능 | 설명 |
|------|------|
| 3패널 대시보드 UI | Primary Source / 다른 관점 / 편향 분석 |
| Agent A: Analyzer | 콘텐츠 파싱, 주장 추출, 에이전트 오케스트레이션 |
| Agent B: Source Verifier | Primary Source 검증, 왜곡 감지 |
| Agent C: Perspective Explorer | 반대 관점 수집, 스펙트럼 매핑 |
| 에이전트 실시간 상태 표시 | 사용자가 에이전트 동작을 볼 수 있음 |
| 최종 분석 카드 | 공유 가능한 결과 카드 |

### 선택 구현 (시간 허용 시)

| 기능 | 설명 |
|------|------|
| Agent D: Socrates | 소크라테스식 대화 (3턴) |
| 분석 히스토리 | 세션 내 임시 저장 |
| 관점 스펙트럼 맵 | 인터랙티브 시각화 |

### 애드온 (데모 후)

| 기능 | 설명 |
|------|------|
| 라이브러리 | 분석 히스토리, 편향 패턴 추적 |
| Chrome Extension | 브라우저에서 바로 분석 |
| 소셜 공유 | 분석 카드 SNS 공유 |
| 다국어 지원 | 영어, 일본어 등 |

## 5. 성공 지표

### 해커톤 심사 기준 대응

| 심사 기준 | 비중 | Flipside 강점 |
|----------|------|---------------|
| Demo | 50% | URL 하나 → 60초 안에 3패널 채워짐. 에이전트 실시간 동작 시연 |
| Impact | 25% | 사회 분열/극단화 직접 공략. 이기려는 동기 이용 → 저항 없음 |
| Creativity | 15% | 미디어 리터러시를 '이기고 싶음'으로 재프레임. 플랫폼 형태 |
| Pitch | 10% | "같은 팩트, 다른 결론" 강한 내러티브 |

### 실패 vs 성공 정의

- **실패**: 사용자가 상대 의견에 **동의**하게 만드는 것
- **성공**: 사용자가 상대 논리를 **이해**하게 만드는 것

> 동의 ≠ 이해. 이해하면서 반대할 수 있습니다. 그게 진짜 비판적 사고입니다.

## 6. 금지 사항 (Anti-Project)

해커톤 금지 항목과 Flipside의 명확한 차이:

| 금지 항목 | Flipside와의 차이 |
|----------|------------------|
| AI 정신건강 어드바이저 | 심리적 지원 X — 논리 분석 도구 |
| 기본 RAG 애플리케이션 | 단순 검색 X — 4개 에이전트 협력 워크플로우 |
| AI for Education 챗봇 | 교육 목적 X — 실제 토론에서 설득하기 위한 실용 도구 |
| 이미지 분석기 | 이미지 분석 자체가 목적 X — 멀티모달은 수단 |
| Streamlit 앱 | 단순 UI X — 3패널 실시간 분석 플랫폼 |

## 7. 리스크 & 미티게이션

### 데모 리스크

| 리스크 | 미티게이션 |
|--------|-----------|
| 에이전트 응답 속도 30초+ | Agent B/C 결과 pre-fetch 후 실시간처럼 스트리밍 |
| Search Grounding 불안정 | 데모용 특정 기사 소스를 사전 캐싱 |
| API 크레딧 소진 | Flash 모델 기본, 핵심 분석만 Pro 사용 |
| UI 렌더링 오류 | 각 패널 독립적 업데이트, graceful fallback |

### 최악의 시나리오 대비
Agent B/C 결과를 하드코딩한 '데모 모드'를 미리 만들어두기. 실제 에이전트가 실패해도 데모는 돌아가야 함.
