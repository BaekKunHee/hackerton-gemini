# Flipside - 해커톤 발표 가이드

> Gemini 3 Seoul Hackathon 2026 | Track: Gemini for Good

---

## 한 줄 소개

**"논쟁에서 이기고 싶으면, 먼저 상대를 이해하세요"**

Flipside는 토론에서 이기고 싶은 사람들을 위한 **비판적 사고 분석 플랫폼**입니다.

---

## 해결하는 사회적 문제

### 1. 사회 양극화와 에코챔버

현대 사회의 심각한 문제:
- **확증 편향**: 자신의 믿음을 확인하는 정보만 소비
- **에코챔버(반향실) 효과**: 같은 생각을 가진 사람들끼리만 소통
- **가짜뉴스 확산**: 팩트체크 없이 감정적으로 공유
- **건강하지 못한 토론 문화**: 상대방 논리를 이해하려 하지 않고 공격만

### 2. 기존 솔루션이 실패하는 이유

기존 미디어 리터러시 도구들의 가정:
> "당신이 편향됐다는 걸 알면, 스스로 고치려 할 것이다"

**이 가정이 틀렸습니다:**

| 기존 접근 | 결과 |
|----------|------|
| "당신이 편향됐어요" | 방어적 반응, 거부감 |
| "팩트체크 하세요" | 귀찮음, 저항 |
| "다른 관점도 봐요" | "왜요?" |

**핵심 문제**: 사람들은 자신이 틀렸다고 느끼는 것을 싫어합니다.

### 3. Flipside의 발상 전환

> 사람들은 **토론에서 이기고 싶어합니다**

이 욕구를 활용하면:
- ✅ 마찰이 없습니다
- ✅ 상대방 논리를 이해하는 것이 **이기기 위한 수단**이 됩니다
- ✅ 그 과정에서 **의도치 않게** 비판적 사고가 작동합니다

**"공부하러 옴" → "이기러 옴"**

## Flipside가 뭐예요?

### 챗봇이 아닙니다. 분석 플랫폼입니다.

| 일반 AI 챗봇 | Flipside |
|-------------|----------|
| 대화창 중심 | **3패널 분석 대시보드** |
| AI 작업 안 보임 | **에이전트 실시간 상태 표시** |
| 텍스트 답변 | **시각화된 분석 카드** |
| 공유 불가 | **분석 카드 공유 가능** |

### 사용 시나리오

1. 사용자가 **논쟁 중인 기사 URL**을 입력합니다
2. 4개의 AI 에이전트가 **실시간으로 분석**합니다
3. 3개의 패널에 결과가 **순차적으로 채워집니다**
4. 소크라테스식 대화로 **자기 생각을 정리**합니다
5. **공유 가능한 분석 카드**를 받습니다

## 핵심 기능: 3개 분석 레이어

### 순서가 중요합니다!

**Layer 3(편향 분석)을 먼저 보여주면 → 방어적 반응**
**Layer 1-2를 먼저 경험하면 → 자연스러운 자기 인식**

### Layer 1: Primary Source 검증
> "이 주장의 근거가 진짜인가?"

- 기사에서 인용한 연구/데이터의 **원본을 자동 탐색**
- 인용이 **정확한지, 왜곡됐는지, 맥락이 누락됐는지** 판정
- 감정이 아닌 **팩트 레벨**에서 시작 → 방어심 낮음

**예시**: "WHO 연구에 따르면..." → 실제 WHO 원문을 찾아 비교

### Layer 2: 다른 관점 탐색
> "같은 사실을 왜 다르게 해석하는가?"

- **같은 팩트, 다른 프레임**으로 보는 관점들 수집
- 관점 스펙트럼 시각화 (진보↔보수, 공포↔희망)
- Layer 1 경험 후라 **'팩트는 같다'가 전제된 상태**

**예시**: 기후변화 → 경제적 관점 vs 환경적 관점 vs 기술 낙관론

### Layer 3: 편향 분석
> "나는 왜 이 관점에 끌렸는가?"

- **Hans Rosling의 10가지 본능** 기반 편향 패턴 진단
- 레이더 차트로 시각화
- Layer 1, 2 경험 후라 **방어적이지 않음**

**10가지 본능 (Factfulness 기반)**:
1. 이분법 본능 (Gap)
2. 부정 본능 (Negativity)
3. 직선 본능 (Straight Line)
4. 공포 본능 (Fear)
5. 크기 본능 (Size)
6. 일반화 본능 (Generalization)
7. 운명 본능 (Destiny)
8. 단일관점 본능 (Single Perspective)
9. 비난 본능 (Blame)
10. 다급함 본능 (Urgency)

## Gemini 3 활용 방법

### 4개의 AI 에이전트

```
입력 (URL/텍스트/이미지)
         │
         ▼
┌─────────────────────────────┐
│   Agent A: Analyzer         │  ← 오케스트레이터 (사용자에게 안 보임)
│   멀티모달 분석, 주장 구조화  │
└─────────────────────────────┘
         │
    ┌────┼────┐
    ▼    ▼    ▼
┌──────┐ ┌──────┐ ┌──────┐
│Agent │ │Agent │ │Agent │
│  B   │ │  C   │ │  D   │
│Source│ │Persp.│ │Socra.│
└──────┘ └──────┘ └──────┘
```

### Gemini 3 기능 활용

| Gemini 기능 | 활용 위치 | 목적 |
|------------|----------|------|
| **Multimodal Vision** | Agent A | 스크린샷/이미지에서 텍스트+맥락 분석 |
| **Deep Think** | Agent A, 편향 분석 | 논리 구조 파악, 다층적 추론 |
| **Google Search Grounding** | Agent B, C | 원본 소스 자율 탐색, 반대 관점 수집 |
| **Structured Output** | 모든 에이전트 | UI 패널용 JSON 생성 |
| **Thought Signatures** | Agent D | 멀티턴 대화 일관성 유지 |

### 왜 Gemini 3인가?

1. **Search Grounding**: 원본 소스를 **자율적으로** 찾아서 검증 (다른 LLM 불가)
2. **Deep Think**: 단순 답변이 아닌 **깊은 추론** 가능
3. **Multimodal**: URL뿐 아니라 **스크린샷도** 바로 분석
4. **Structured Output**: UI에 바로 렌더링할 수 있는 **정확한 JSON** 출력

## 서비스 장점

### 1. 심리적 저항 없음
- "당신이 틀렸어요" ❌
- "이기려면 상대를 알아야죠" ✅

### 2. 실시간 AI 가시성
- 4개 에이전트가 뭘 하는지 **실시간으로 보여줌**
- "AI가 뭘 하는지 모르겠어요" 문제 해결
- 신뢰도 상승

### 3. 공유 가능한 결과물
- 분석 카드를 **논쟁 상대에게 공유** 가능
- "내가 이렇게 분석했는데, 어떻게 생각해?"

### 4. Steel-Manning 접근
- 상대 논리를 **가장 강한 버전**으로 이해
- 허수아비 논증(Straw Man) 방지
- **진짜 대화**의 시작점

### 5. 순차적 공개 설계
- 편향 분석을 **마지막에** 보여줌
- 팩트 → 관점 → 편향 순서로 **자연스러운 인식**

## 데모 시나리오 (60초)

### 준비물
- 논쟁적인 뉴스 기사 URL (예: 기후변화, 경제정책 등)

### 데모 흐름

**[0-10초]** URL 입력
> "요즘 논쟁 중인 기후변화 기사 하나를 분석해볼게요"

**[10-30초]** 에이전트 실시간 동작
> "보시면 4개의 AI 에이전트가 동시에 작업하고 있어요.
> Source Verifier가 인용된 연구 원본을 찾고 있고,
> Perspective Explorer가 다른 관점의 기사를 수집하고 있습니다."

**[30-45초]** 3패널 결과
> "왼쪽 패널: 이 기사가 인용한 소스가 실제로 어떻게 말했는지
> 가운데 패널: 같은 주제를 다른 관점에서 본 기사들
> 오른쪽 패널: 이 기사에서 작동하는 인지 편향 패턴"

**[45-60초]** Steel Man 카드
> "최종 결과는 공유 가능한 분석 카드예요.
> 상대 논리의 가장 강한 버전, 그리고 내 주장을 강화할 포인트.
> 이걸 논쟁 상대에게 보내면 진짜 대화가 시작됩니다."

## 핵심 메시지 (심사위원용)

### Impact (25%)
- **사회 분열/극단화** 직접 공략
- 기존 미디어 리터러시 도구의 **근본적 한계** 해결
- **"이기고 싶음"**을 이용 → 저항 없는 비판적 사고

### Demo (50%)
- URL 하나 → **60초 안에 3패널** 채워짐
- 에이전트 **실시간 동작** 시연
- 공유 가능한 **분석 카드** 결과물

### Creativity (15%)
- 미디어 리터러시를 **"이기고 싶음"**으로 재프레임
- 챗봇이 아닌 **플랫폼** 형태
- **Steel-Manning** 철학 적용

### Technical (Gemini 활용)
- **Search Grounding**: 원본 소스 자율 탐색 (핵심 차별점)
- **Deep Think**: 편향 패턴 분석
- **Multimodal**: 스크린샷 분석
- **4-Agent 협력 워크플로우**: 단순 RAG 아님

## 자주 나올 질문 (Q&A 대비)

### Q: 이거 그냥 팩트체크 서비스 아닌가요?
> "팩트체크는 '틀렸다/맞다'를 판정합니다. Flipside는 **왜 같은 팩트를 다르게 해석하는지** 보여줍니다. 목적이 다릅니다."

### Q: 사람들이 실제로 쓸까요?
> "핵심은 **진입 동기**입니다. '내 편향 확인하기'로는 안 씁니다. '토론에서 이기기'로 프레임하면 저항 없이 씁니다."

### Q: 기존 미디어 리터러시 도구와 뭐가 다른가요?
> "기존 도구: '당신이 편향됐어요' → 방어적 반응
> Flipside: '이기려면 상대를 알아야죠' → 자발적 참여"

### Q: AI가 편향을 분석하면 그것도 편향 아닌가요?
> "맞습니다. 그래서 Flipside는 **Hans Rosling의 10 Instincts**라는 학문적 프레임워크를 사용합니다. AI의 주관이 아니라 인지과학 기반 분석입니다."

### Q: 단순 RAG 아닌가요?
> "RAG는 검색 → 답변입니다. Flipside는 4개 에이전트가 **역할 분담**하고 **협력**합니다. 분석 → 검증 → 관점 탐색 → 대화가 **워크플로우**로 연결됩니다."

## 성공의 정의

### 실패
사용자가 상대 의견에 **동의**하게 만드는 것

### 성공
사용자가 상대 논리를 **이해**하게 만드는 것

> **동의 ≠ 이해**
> 이해하면서 반대할 수 있습니다.
> 그게 진짜 비판적 사고입니다.
